🛒 dp-Store - MVP Summary

A full-stack e-commerce website with payment integration

📌 Project Overview

dp-Store is a modern full-stack e-commerce MVP inspired by eBay. It includes a user-facing shopping frontend with secure payments, a robust Express backend API, and a powerful admin dashboard for managing users, products, orders, and transactions.

⚙️ Tech Stack

  • Frontend: Next.js + Tailwind CSS + Zustand
  • Admin Panel: Next.js + Zustand + React Hook Form
  • Backend: Express.js + TypeScript + Firebase
  • Database: Firebase Firestore
  • Payments: Stripe API integration
  • Image Upload: Cloudinary
  • Auth: Firebase (Google, GitHub, Email)
  • Deployment: Vercel (frontend), Google Cloud (backend)
  • Tools: Docker, Railway, FileSaver, CSV Export
  • Payment Testing: Stripe test environment

✅ Features

User Features
  • 🛍️ Browse products by category
  • 💖 Wishlist & 🛒 Cart functionality
  • 🔐 Auth via Google, GitHub, Email
  • 💳 Secure checkout with Stripe payment processing
  • 📧 Order confirmation emails
Admin Features
  • 📦 Product CRUD with Cloudinary images
  • 👥 User management (promote/delete)
  • 🧾 CSV export, search, pagination
  • 💰 Transaction monitoring & refund processing
  • 🪵 Activity log in Firestore

💳 Payment Integration

Stripe Implementation
  • 🔒 Secure client-side checkout flow
  • 📝 Order tracking with Firestore
  • 🔄 Webhook for payment status updates
  • 🧪 Test mode with dummy cards
Payment Features
  • 💵 Multiple currency support
  • 📊 Sales analytics dashboard
  • 🔄 Refund processing
  • 📄 Receipt generation

🚀 Deployment Journey

  • ✅ Local testing of all modules (frontend + backend + payments)
  • ⚠️ Railway: CI/CD failures and broken URL issues
  • ✅ Dockerized backend locally for testing
  • ✅ Stripe webhook endpoint configuration
  • ✅ Final deployment on Google Cloud Compute with static IP + firewall config
  • ✅ Frontend + Admin hosted via Vercel

🗓️ 5-Day MVP Timeline

  • Day 1: Project setup, Firebase Auth, planning, and initial Stripe configuration
  • Day 2: Core frontend development, basic product CRUD, and shopping cart with Stripe elements
  • Day 3: Backend API, admin dashboard, and payment webhook integration
  • Day 4: Complete checkout flow, UI polish, Cloudinary integration, and transaction logging
  • Day 5: Deployment preparation, testing (including payment test mode), and final deploy to Google Cloud

🙌 Final Note

This MVP project demonstrates full-stack architecture with secure payment processing, deployment challenges, admin tooling, UI polish, and async communication skills. The Stripe integration provides a production-ready payment solution with test mode capabilities during development.
